# Apollo Tuist Bootstrap

This bootstrap introduces a parallel Apple-platform structure without disturbing the existing CMake tree.

## Targets

- `ApolloApp`
  Swift app shell generated by Tuist.
- `ApolloMacBridge`
  Swift bridge target that owns Apple-platform orchestration and will later host the `MacDisplayKit` integration.
- `ApolloCore`
  C/C++ compatibility surface for the existing Apollo runtime.
- `ApolloTuistTests`
  Basic validation for the Swift-to-core boundary.

## Responsibility Split

- `ApolloApp`
  UI shell, lifecycle, and future Swift-only orchestration.
- `ApolloMacBridge`
  Apple-specific bridge surface, callback/session adapters, and the eventual `MacDisplayKit` hook-up.
- `ApolloCore`
  Existing C/C++ Apollo logic exposed through a stable C ABI.

## Next Slice

1. Add an ObjC++ callback bridge from `ApolloMacBridge` into Apollo's native C++ encode/session runtime.
2. Move the current macOS encoded-frame consumer out of the legacy Objective-C runtime and behind the bridge target.
3. Reuse the local `MacDisplayKit` package dependency for callback-only capture startup and native Apollo adapter wiring.
